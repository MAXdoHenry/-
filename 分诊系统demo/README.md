# 分诊系统演示版

这是一个基于轻量后端、单文件数据库和网页模板的医联体分级会诊/分诊演示系统。

项目目标是演示：

- 医生登录
- 发起疑难病例提问
- 同级、上级、跨级医院请教
- 附件上传
- 会诊回复
- 回复通知
- 按可见医院等级控制查看权限
- 手机号/微信号添加好友
- 一对一私聊、群聊、群聊附件、病例聊天室
- 医疗敏感内容提示和操作日志留痕

本项目是教学和演示用系统，不包含真实医疗数据。

## 1. 技术栈

- 后端：Python Flask
- 数据库：SQLite
- 前端：HTML + Bootstrap 5 CDN
- 附件：保存到本地 `uploads/` 文件夹

## 3. 如何运行

如果只是访问别人已经部署好的 Demo，只需要浏览器，不需要安装 Python 或 Flask。

如果是从 GitHub 下载源码，并且想在自己的电脑上运行 Demo，需要先安装 Python 3。

进入项目目录：

```bash
cd <项目目录>
```

安装依赖：

```bash
pip install flask
```

启动系统：

```bash
python app.py
```

启动成功后，终端会看到类似输出：

```text
Running on http://127.0.0.1:5000
```

本机浏览器访问：

```text
http://127.0.0.1:5000
```

如果要让同一局域网内的其他电脑或手机访问，请使用部署电脑的局域网 IP：

```text
http://<部署电脑IP>:5000
```

例如：

```text
http://192.168.x.x:5000
```

## 5. 功能说明

### 登录

访问系统后会先进入登录页。

未登录用户访问其他页面时，会自动跳回登录页。

### 首页

登录后进入首页，展示：

- 当前医生信息
- 我的提问数量
- 待回复数量
- 已回复数量
- 最新提问列表

### 发起提问

医生可以填写：

- 标题
- 病情描述
- 请教医院
- 附件
- 哪些等级医院可以查看此提问

请教医院支持同级、上级、跨级选择。

### 附件上传

提问、回复和群聊消息都支持附件。

当前支持：

```text
png、jpg、jpeg、gif、webp、pdf、mp4、mov、avi、webm
```

图片和视频会在页面中预览，其他附件提供下载。

### 我的提问

只显示当前登录医生自己发起的提问。

### 会诊提问列表

显示当前医生有权限查看的其他提问。

可见规则：

1. 自己发起的提问永远可见。
2. 其他提问是否可见，由提问人选择的“可见医院等级”决定。
3. 被指定为“请教医院”的医生可以查看并回复。
4. 只有被请教医院的医生可以回复，其他可见医院只能查看。

### 提问详情

详情页展示：

- 提问标题
- 提问人
- 所属医院
- 科室
- 提问时间
- 病情描述
- 附件
- 可见医院等级
- 回复内容

### 回复通知

当自己的提问被回复后，页面顶部会显示：

```text
您的提问已被回答
```

### 好友与聊天

系统新增通用好友体系：

- 输入手机号或微信号搜索医生。
- 发送好友申请。
- 对方可同意或拒绝。
- 好友可按分组管理。
- 好友资料统一显示姓名、医院、科室、职称，不支持昵称。

系统新增医疗聊天功能：

- 好友之间可以一对一私聊。
- 可以任意选择好友创建群聊。
- 群聊消息支持上传图片、视频和文档文件。
- 群聊可以只上传附件，也可以文字和附件一起发送。
- 群聊类型包括病例讨论群、科室工作群、学术交流群、培训群。
- 私聊和群聊都可以一键发起病例会诊。
- 病例详情页可以进入病例聊天室，也可以邀请好友会诊。

医疗合规设计：

- 聊天消息不提供删除或撤回按钮。
- 关键操作写入 `operation_logs`。
- 涉及诊断、用药、治疗方案等内容时，页面会提示“本交流仅作临床参考，不作为正式诊断依据”。

## 6. 如何验证 Demo

下面是一组推荐验证流程。

### 第一步：验证系统能启动

执行：

```bash
python app.py
```

浏览器访问：

```text
http://127.0.0.1:5000
```

如果能看到登录页，说明服务启动成功。

### 第二步：验证登录

使用以下账号登录：

```text
医生姓名：赵医生
密码：123456
```

登录后如果进入首页，说明登录功能正常。

### 第三步：验证发起提问

用 `赵医生` 登录后，进入“发起提问”页面。

填写：

```text
标题：测试会诊问题
病情描述：这是一个测试病例，用于验证 Demo 是否正常。
请教医院：任选一个医院
```

可选上传一张图片或一个 PDF。

点击“提交提问”。

如果跳转到“我的提问”，并且能看到刚提交的记录，说明提问功能正常。

### 第四步：验证附件

进入刚才提交的提问详情页。

如果上传的是图片或视频，应能看到预览。

如果上传的是 PDF 或其他支持文件，应能看到下载按钮。

### 第五步：验证回复

退出当前账号。

使用被请教医院的医生账号登录，例如：

```text
医生姓名：张主任
密码：123456
```

进入“会诊提问列表”，找到刚才的提问，点击查看详情。

填写回复内容并提交。

如果状态变为“已回复”，说明回复功能正常。

### 第六步：验证回复通知

退出当前账号。

重新登录原提问人：

```text
医生姓名：赵医生
密码：123456
```

如果页面顶部显示“您的提问已被回答”，说明通知功能正常。

### 第七步：验证权限

可以分别登录不同医生账号观察：

- 自己只能在“我的提问”中看到自己发起的提问。
- “会诊提问列表”只显示当前账号有权限查看的提问。
- 只有被指定为请教医院的医生可以回复。

### 第八步：验证好友和聊天

用 `赵医生` 登录，进入“好友”页面。

点击“添加好友”，输入：

```text
2
```

可以搜索到 `李医生`。

如果两人已经是演示好友，可以直接进入私聊；否则发送好友申请后，换对方账号到“新的朋友”里处理。

进入私聊后，发送包含以下词语的消息：

```text
诊断
用药
治疗方案
```

页面会弹出医疗风险提示，消息下方也会显示临床参考说明。

### 第九步：验证群聊和病例聊天室

进入“群聊”页面：

- 可以查看演示群。
- 可以新建群聊。
- 可以邀请好友入群。
- 群聊内可以发送消息。
- 群聊内可以上传附件，也可以只上传附件不写文字。
- 群聊内可以点击“发起病例会诊”。

进入任意病例详情页：

- 点击“进入病例聊天室”。
- 点击“邀请好友会诊”。

如果好友被邀请进入病例聊天室，该好友也可以查看对应病例内容。

### 第十步：验证群聊附件

进入任意群聊页面。

在底部发送区域选择一个图片、视频或文档文件。

可以不填写文字，直接点击“发送”。

发送后：

- 图片和视频会在聊天消息中预览。
- 其他文件会显示“下载附件”按钮。
- 只有该群成员可以下载群聊附件。
